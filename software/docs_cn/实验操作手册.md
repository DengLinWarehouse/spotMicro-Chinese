# Spot Micro 软件实验操作手册

> 适用对象：第一次在 Ubuntu + ROS 环境中复刻 Spot Micro 软件栈、希望“照着做”完成一次从编译到实机联调的同学。如果你对 ROS 或 Linux 毫无经验，请严格按照步骤执行，遇到未覆盖的情况再查阅官方文档。

## 0. 实验阶段划分
1. **环境确认**：准备硬件、安装 ROS、配置 catkin 工作区。
2. **软件构建**：同步仓库、安装依赖、完成 `catkin build`。
3. **舵机校准**：使用 `servo_move_keyboard` 逐个记录 PWM ↔ 角度，对应 `docs_cn/servo_calibration.md`。
4. **系统联调**：依次启动驱动、遥控、运动控制、可视化，最终让机器人稳定站立并走步。

每一阶段结束都要记录：执行命令、终端输出、是否达到预期。没有记录的步骤视为“未完成”。

## 1. 环境与前置条件

### 1.1 推荐硬件
- 主控：树莓派 4B (4GB) 或 Jetson Nano（仅需跑 ROS 节点即可）。
- 电源：通过 BEC 输出 5V/5A 给树莓派与 PCA9685；舵机供电需要独立 7.4V LiPo 电池 + 5A 以上带保险丝的电源线。
- 控制板：PCA9685 + I2C 接口 + 杜邦线；可选 IMU、Lidar（若要跑 SLAM）。
- 计算机：一台 Ubuntu 20.04 x64 PC 用于开发（安装 ROS Noetic）。

> **安全提示**：舵机供电必须与主控共地；所有动力线在合体之前先用万用表确认无短路。

### 1.2 软件环境
- Ubuntu 20.04 + ROS Noetic (Desktop-Full)。
- Python 3.8、自带 `pip`、`virtualenv`。
- Catkin 工具：`sudo apt install python3-catkin-tools`。
- 依赖（一次执行）：
  ```bash
  sudo apt update
  sudo apt install git build-essential cmake python3-rosdep python3-vcstool libeigen3-dev \
       ros-noetic-rosserial ros-noetic-joy ros-noetic-tf ros-noetic-diagnostic-updater \
       ros-noetic-xacro ros-noetic-rviz ros-noetic-ros-control ros-noetic-ros-controllers \
       python3-smbus python3-matplotlib
  sudo rosdep init   # 仅第一次
  rosdep update
  ```

### 1.3 仓库与目录
- 建议目录：`~/catkin_ws/src/spotMicro-Chinese/software`。
- 确认 `software/` 下已是完整镜像（可运行 `python scripts/compare_with_upstream.py`，输出 `Missing from 软件部分: 0`）。

## 2. Catkin 工作区配置

1. 创建工作区：
   ```bash
   mkdir -p ~/catkin_ws/src
   cd ~/catkin_ws
   catkin init
   catkin config --extend /opt/ros/noetic --cmake-args -DCMAKE_BUILD_TYPE=Release
   ```
2. 将仓库链接到 `src`：
   ```bash
   cd ~/catkin_ws/src
   ln -s /path/to/spotMicro-Chinese/software spot_micro
   ```
3. 安装依赖并构建：
   ```bash
   cd ~/catkin_ws
   rosdep install --from-paths src --ignore-src -r -y
   catkin build
   source ~/catkin_ws/devel/setup.bash
   ```
4. **验证点**：终端最后出现 `==> Finished processing package [spot_micro_motion_cmd]` 等全部成功；`echo $ROS_PACKAGE_PATH` 中包含 `spot_micro`。

若构建失败：
- 查看 `/catkin_ws/build/*/CMakeFiles/CMakeOutput.log`。
- 常见原因：缺少 `libeigen3-dev`、`python3-smbus`、或 `rosdep` 未初始化。

## 3. 舵机校准快速流程

详尽解释见 `docs_cn/servo_calibration.md`，这里给出实操顺序：

1. **断电装配**：全部舵机初始位置约为中位，连结各连杆，注意不要拧紧自攻螺丝以便调节。
2. **启动驱动**（树莓派）：
   ```bash
   rosrun i2cpwm_board i2cpwm_board
   ```
3. **键盘调试**（PC 或树莓派均可，需 `ROS_MASTER_URI` 指向驱动机）：
   ```bash
   rosrun servo_move_keyboard servoMoveKeyboard.py
   ```
4. 选择 `oneServo` → 输入 PCA9685 通道号 → 使用 `y/g/j/f/k/t/u` 微调，记录中心值与±参考角度 PWM。
5. 将测得的 PWM+角度填入 `docs_cn/舵机校准参考表格.ods`，复制生成的参数到 `spot_micro_motion_cmd/config/spot_micro_motion_cmd.yaml` 中对应字段。
6. **再次通电测试**：运行 `rosrun spot_micro_motion_cmd spot_micro_motion_cmd_node`，确认机器人能站稳；若抖动，检查对应 Servo `direction` 与 `center_angle_deg`。

> **高危操作提示**：任何修改 `spot_micro_motion_cmd.yaml` 后必须先架空机器人再上电，待确认运动方向正确后再落地。

## 4. 实验执行步骤

| 阶段 | 命令 | 期望现象 | 常见错误及排查 |
| --- | --- | --- | --- |
| 4.1 启动 `roscore` | `roscore` | 终端显示 `started core service [/rosout]` | 如果端口冲突，检查是否已有 roscore，使用 `rosnode list` 确认 |
| 4.2 启动 PCA9685 驱动 | `rosrun i2cpwm_board i2cpwm_board`（在树莓派） | 终端每 1s 打印 `Publishing servo feedback` | 若出现 `Cannot open /dev/i2c-1`：确认 I2C 已启用（`sudo raspi-config` → Interfacing Options → I2C）|
| 4.3 舵机动作自检 | `rosrun servo_move_keyboard servoMoveKeyboard.py` | `oneServo` 模式下舵机可响应 PWM 改变 | 若报 `topic [/servos_absolute] does not exist`：检查驱动节点是否运行，网络是否连通 |
| 4.4 键盘遥控 | `rosrun spot_micro_keyboard_command spotMicroKeyboardMove.py` | 新窗口显示按键映射 (`W/S` 前进后退等)，按键后 `cmd_vel` 话题在 `rostopic echo` 中变化 | 若提示 `can't find launch file`，先运行 `source ~/catkin_ws/devel/setup.bash` |
| 4.5 启动运动控制 | `rosrun spot_micro_motion_cmd spot_micro_motion_cmd_node` | 终端输出状态机日志：`Entering state: Stand`→`Walk`，机器人站立 | 若抖动或倒地：立即按急停/断电，并回到舵机校准阶段 |
| 4.6 可视化 | `roslaunch spot_micro_rviz show_and_move_model_via_gui.launch` | RViz 展示机器人模型，跟随 `cmd_vel` 改变姿态 | 若 URDF 报错：确认 `larm.stl` 等文件存在，必要时 `catkin clean spot_micro_rviz && catkin build` |
| 4.7 全流程 | `roslaunch spot_micro_launch keyboard_control_and_rviz.launch` | 所有关节点一次启动；终端无错误并进入 `stand` 状态 | 若 `spot_micro_motion_cmd` 报无法连接 `servos_absolute`，检查 `ROS_NAMESPACE` 和网络配置 |

**网络配置**：
- 在树莓派（作为 ROS master）设置：
  ```bash
  export ROS_MASTER_URI=http://<pi-ip>:11311
  export ROS_HOSTNAME=<pi-ip>
  ```
- 在 PC（作为节点）设置：
  ```bash
  export ROS_MASTER_URI=http://<pi-ip>:11311
  export ROS_HOSTNAME=<pc-ip>
  ```
- 使用 `ping <对方IP>`、`rosnode list` 验证连通。

**安全顺序**：
1. 上电顺序：主控 → 传感器 → 舵机电源（最后）。
2. 下电顺序：舵机电源 → 传感器 → 主控。
3. 实验过程中保留实体急停或随时拔掉舵机电源的开关。

## 5. 常见问题排查

| 现象 | 可能原因 | 处理办法 |
| --- | --- | --- |
| `catkin build` 无法找到 `spot_micro_joy` | 未在 `src` 中创建软链接或 clone 完整仓库 | 确认 `~/catkin_ws/src/spot_micro/spot_micro_joy/package.xml` 存在 |
| 舵机只向一个方向动且抖动 | `direction` 设置错误或供电不足 | 调换 `direction` 的 ±1；检查电池电压保持 7–8V |
| `rosrun spot_micro_motion_cmd ...` 提示 `Failed to connect to i2cpwm_board` | 未运行驱动或网络变量错误 | 重启 `i2cpwm_board`，确认 `ROS_MASTER_URI` 指向驱动机 |
| RViz 模型错位 | URDF 依赖的 STL 未安装 / 未生成 TF | 重新 `catkin build spot_micro_rviz`，并确保 `spot_micro_motion_cmd` 发布 TF |
| 键盘节点无响应 | 终端焦点不在键盘窗口；或未安装 `python3-evdev`（在树莓派上） | 切换到终端窗口；`sudo apt install python3-evdev` |
