# 舵机标定：从物理测量到 YAML 参数的完整计算过程

本文档详细记录了 SpotMicro 四足机器人 12 个舵机的标定工程全过程，包括：物理测量方法、YAML 参数的数学推导、IK（逆运动学）源码分析、以及最终结果的正确性验证。适用于使用 **PDI-HV5523MG**（120° / 900–2100 µs）舵机的用户。

> **前置阅读**：本文假设你已阅读 `servo_calibration.md`（标定操作指南），了解 `servo_move_keyboard` 工具的基本使用方法。

## 1. 背景知识

### 符号约定

本文统一使用以下符号，与 YAML 参数名的对应关系如下：

| 符号 | YAML / 代码参数 | 含义 |
|:---:|:---|:---|
| $\theta_{\text{IK}}$ | `cmd_ang_rad` | IK 逆运动学输出的关节角度 |
| $C$ | `center` | 舵机中心参考点的 PWM 值 |
| $R$ | `range` | 覆盖 $\pm\theta_{\max}$ 的 PWM 总跨度 |
| $d$ | `direction` | 旋转方向，$+1$ 或 $-1$ |
| $\theta_{\text{center}}$ | `center_angle_deg` | 舵机处于 $C$ 时对应的关节角度 |
| $\theta_{\max}$ | `servo_max_angle_deg` | 单侧最大命令角度（本项目 = 55.0°） |
| $p$ | proportional | 归一化比例命令，$\in [-1,\;+1]$ |
| $k$ | slope | 线性标定斜率（PWM/°） |
| $a$ | intercept | 线性标定截距（= $\text{PWM}_{0°}$） |

### 1.1 角度到 PWM 的映射链路

机器人运动控制的完整链路如下：

```
IK 逆运动学  →  关节角度(rad)  →  proportional 比例命令  →  i2cpwm_board  →  PWM 脉冲  →  舵机转动
```

其中，关节角度到 PWM 的转换由两步完成：

**第一步**（`spot_micro_motion_cmd.cpp` 第 253–256 行 / `servo_driver.py` 第 24 行）：

$$
p = \frac{\theta_{\text{IK}} - \theta_{\text{center}}}{\theta_{\max}}
$$

其中 $p$ 为 proportional 比例命令，$\theta_{\text{IK}}$ 为 IK 输出的关节角度，$\theta_{\text{center}}$ 为 `center_angle_deg`，$\theta_{\max}$ 为 `servo_max_angle_deg`。该值被裁剪（clamp）到 $[-1,\;+1]$ 范围。

**第二步**（`i2cpwm_board` 驱动内部 / `servo_driver.py` 第 27–28 行）：

$$
\text{PWM} = C + d \cdot \frac{R}{2} \cdot p
$$

其中 $C$ 为 `center`，$d$ 为 `direction`（$\pm 1$），$R$ 为 `range`。

合并后得到**完整公式**：

$$
\text{PWM} = C + d \cdot \frac{R}{2} \cdot \frac{\theta_{\text{IK}} - \theta_{\text{center}}}{\theta_{\max}}
$$

### 1.2 YAML 配置参数含义

以 `spot_micro_motion_cmd.yaml` 中一行为例：

```yaml
LF_1: {num: 1, center: 306, range: 196, direction: 1, center_angle_deg: -6.8}
```

| 参数 | 含义 |
|:---|:---|
| `num` | PCA9685 上的舵机端口号（1–16） |
| `center` | 舵机"中心参考点"的 PWM 值（通常为 306 ≈ 1500 µs） |
| `range` | 覆盖 ±`servo_max_angle_deg` 的 PWM 总跨度 |
| `direction` | +1 或 −1，是否翻转 PWM 与角度的对应方向 |
| `center_angle_deg` | 舵机处于 `center` PWM 时，对应的关节角度（IK 坐标系） |

$\theta_{\max}$（即 `servo_max_angle_deg`，本项目设为 55.0°）定义了 $p = \pm 1$ 对应的角度偏差。

### 1.3 PDI-HV5523MG 关键参数

| 参数 | 值 |
|:---|:---|
| 物理行程 | 120°（中心左右各 60°） |
| 脉宽范围 | 900–2100 µs |
| PCA9685 PWM 范围 | 184–430（50 Hz 频率下） |
| `servo_max_angle_deg` | 55.0°（留 5° 安全余量） |

## 2. 物理测量：两点标定法

### 2.1 标定原理

舵机的角度–PWM 关系在有效行程内近似线性。只需测量**两个已知角度**对应的 PWM 值，即可确定整条直线，从而反算出全部 4 个 YAML 参数。

### 2.2 角度约定

各关节的 **0° 参考位置**和**正方向**定义如下：

| 关节类型 | 0° 定义 | 正方向 | 标定测量角度 |
|:---|:---|:---|:---|
| 肩关节（Link 1） | 腿既不内旋也不外旋，处于自然垂直平面 | 左腿外旋为正，右腿外旋为负 | 0° 和 ±45° |
| 大腿（Link 2） | 大腿自然垂直向下 | 左腿向后抬为正，右腿向后抬为负 | 0° 和 ±90° |
| 小腿（Link 3） | 小腿与大腿成一条直线（完全伸直） | 左腿向前弯为负，右腿向前弯为正 | 0° 和 ±90° |

> **关键点**：上述角度约定与 IK 代码中的关节角度坐标系完全一致（详见第 3 节的代码分析），这是计算正确的前提。

### 2.3 操作步骤

1. **架空机器人**，让四条腿自由下垂
2. 启动 `i2cpwm_board` 和 `servo_move_keyboard`
3. 输入 `oneServo`，选择要标定的舵机编号（1–12）
4. 用 `g/j`（步进 ±1）和 `f/k`（步进 ±10）微调 PWM
5. 将关节调到 **0° 位置**，记录终端显示的 PWM 值
6. 再调到**第二个参考角度**（±45° 或 ±90°），记录 PWM 值
7. 对全部 12 个舵机重复上述过程

## 3. IK 源码分析：验证角度约定的一致性

在计算 YAML 参数之前，必须确认标定时的物理角度约定与 IK 代码的关节坐标系一致。以下是对源码的关键分析。

### 3.1 腿的分类（`spot_micro_kinematics.cpp` 第 30–33 行）

```cpp
right_back_leg_  = SpotMicroLeg(joint_angles_temp, link_lengths_temp, true);   // is_leg_12 = true
right_front_leg_ = SpotMicroLeg(joint_angles_temp, link_lengths_temp, true);   // is_leg_12 = true
left_front_leg_  = SpotMicroLeg(joint_angles_temp, link_lengths_temp, false);  // is_leg_12 = false
left_back_leg_   = SpotMicroLeg(joint_angles_temp, link_lengths_temp, false);  // is_leg_12 = false
```

- `is_leg_12 = true`：**右腿**（RF, RB）
- `is_leg_12 = false`：**左腿**（LF, LB）

### 3.2 逆运动学输出角度的符号（`utils.cpp` 第 177–219 行）

```cpp
if (is_leg_12) {
    joint_angles.ang3 = atan2( sqrt(1 - D*D), D);   // 右腿: ang3 ∈ [0, +π]
} else {
    joint_angles.ang3 = atan2(-sqrt(1 - D*D), D);   // 左腿: ang3 ∈ [-π, 0]
}
```

即：

$$
\text{ang3} =
\begin{cases}
\arctan2\!\bigl(\sqrt{1-D^2},\; D\bigr) \in [0,\;+\pi] & \text{右腿 (RF/RB)} \\[4pt]
\arctan2\!\bigl(-\sqrt{1-D^2},\; D\bigr) \in [-\pi,\;0] & \text{左腿 (LF/LB)}
\end{cases}
$$

| 关节 | 右腿 (RF/RB) | 左腿 (LF/LB) |
|:---|:---|:---|
| ang3（小腿） | $[0°,\;+180°]$，弯曲为正 | $[-180°,\;0°]$，弯曲为负 |

### 3.3 ang3 = 0 的物理含义

从正运动学链（`ht0To4`）分析：当 $\theta_1 = \theta_2 = \theta_3 = 0$ 时，足端位置为 $(-l_1,\;-(l_2+l_3),\;0)$，即腿完全伸直向下。因此：

- $\theta_3 = 0$ ↔ 小腿与大腿成一条直线 ↔ 标定的物理 0° ✓
- $\theta_2 = 0$ ↔ 大腿自然垂直向下 ↔ 标定的物理 0° ✓

> **结论**：标定时的物理角度定义与 IK 代码的关节坐标系完全一致，可以直接用物理测量角度进行计算。

## 4. 数学推导：从两点数据到四个 YAML 参数

### 4.1 线性模型

两个测量点 $(\theta_A,\;\text{PWM}_A)$ 和 $(\theta_B,\;\text{PWM}_B)$ 确定一条直线：

$$
\text{PWM} = a + k\,\theta
$$

其中：

$$
k = \frac{\text{PWM}_B - \text{PWM}_A}{\theta_B - \theta_A} \qquad \text{（斜率，单位：PWM/°）}
$$

$$
a = \text{PWM}_A - k \cdot \theta_A \qquad \text{（截距，即 } \theta = 0° \text{ 时的 PWM）}
$$

### 4.2 与 YAML 公式对齐

将完整映射公式展开为 $\theta$ 的一次函数：

$$
\text{PWM} = C + d \cdot \frac{R}{2} \cdot \frac{\theta - \theta_{\text{center}}}{\theta_{\max}}
$$

令 $\displaystyle K = \frac{d \cdot R}{2\,\theta_{\max}}$，则：

$$
\text{PWM} = \bigl(C - K\,\theta_{\text{center}}\bigr) + K\,\theta
$$

对比线性模型 $\text{PWM} = a + k\,\theta$，可得到等价关系：

$$
K = k \;, \qquad C - K\,\theta_{\text{center}} = a
$$

### 4.3 求解四个参数

**① direction**（旋转方向）：

$$
d = \operatorname{sign}(k)
$$

- $k > 0 \Rightarrow d = +1$（PWM 随角度增大而增大）
- $k < 0 \Rightarrow d = -1$（PWM 随角度增大而减小）

**② range**（PWM 跨度）：

$$
R = |k| \times 2\,\theta_{\max} = |k| \times 2 \times 55.0 = |k| \times 110
$$

**③ center**（中心 PWM）：

采用项目惯例 $C = 306$（≈ 1500 µs，舵机物理中位）。

**④ center_angle_deg**（中心角度偏移）：

由 $C - K\,\theta_{\text{center}} = a$ 推出：

$$
\theta_{\text{center}} = \frac{C - a}{k} = \frac{306 - \text{PWM}_{0°}}{k}
$$

其物理意义：当舵机处于 PWM 306 时，关节实际处于 $\theta_{\text{center}}$ 角度。

### 4.4 完整计算示例

以 **舵机 2（LF_2 左前大腿）** 为例，测量数据：$0° \to \text{PWM}\;176$，$+90° \to \text{PWM}\;378$。

**第 1 步 — 斜率：**

$$
k = \frac{378 - 176}{90 - 0} = \frac{202}{90} = 2.244
$$

**第 2 步 — direction：**

$$
d = \operatorname{sign}(2.244) = +1
$$

**第 3 步 — range：**

$$
R = 2.244 \times 110 = 246.9 \;\approx\; 247
$$

**第 4 步 — center_angle_deg（** $a = 176$，即 $\text{PWM}_{0°}$ **）：**

$$
\theta_{\text{center}} = \frac{306 - 176}{2.244} = \frac{130}{2.244} = 57.9°
$$

**验证**：当 IK 命令角度 $\theta_{\text{IK}} = 0°$（大腿垂直）：

$$
p = \frac{0° - 57.9°}{55°} = -1.053 \;\xrightarrow{\text{clamp}}\; -1.0
$$

$$
\text{PWM} = 306 + 1 \times \frac{247}{2} \times (-1.0) = 306 - 123.5 = 182.5
$$

实测 0° 对应 PWM 176，误差约 6.5 PWM（因为 $p$ 被轻微裁剪），在可接受范围内。

对于正常的站立和行走（小腿角度约 60°–130°），这些裁剪极少触发。若在 ROS 日志中看到 `Proportional Command above/below ±1.0 was computed, clipped` 警告，说明某个关节正在被裁剪，可考虑重新安装舵盘以改善 0° 位置对应的 PWM 值。

## 8. 常见问题

### Q1: center_angle_deg 为什么有些值很大（如 121.8°）？

`center_angle_deg` 表示舵机在 PWM 306（物理中位）时的关节角度。如果舵盘安装得使 0° 位置（如小腿伸直）远离舵机中位，该值就会很大。这在数学上完全正确，不影响运行。

### Q2: 所有舵机的 direction 都是 +1，正常吗？

direction 取决于舵机的物理安装方向。如果所有舵机在角度增大时 PWM 都增大（slope > 0），那么 direction 全为 +1 是正确的。原项目中 RF_1 和 LB_1 为 −1 是因为原作者的舵机安装方向不同。

### Q3: 肩关节（Link 1）的 range 为什么明显小于大腿和小腿？

肩关节只测量了 0° 到 ±45°（45° 跨度），而大腿和小腿测量了 0° 到 ±90°（90° 跨度）。较小的测量跨度放大了角度误差对斜率的影响，导致 range 的计算偏差较大。如需更精确，可尝试更大角度的测量（如 ±60°，前提是机械结构允许）。

### Q4: 舵机 6 和 12 的 0° PWM 值（76 和 66）远低于安全范围（184），会有问题吗？

这表示舵盘安装位置偏极端——小腿伸直时舵机已接近物理行程极限。在正常运行中，小腿很少完全伸直（IK 角度 ≈ 60°–130°），此时 PWM 在安全范围内。但如果行走过程中出现小腿接近伸直的情况，建议重新安装舵盘使 0° 位置更靠近 PWM 200–300 区间。

### Q5: 标定后需要重新编译 catkin_ws 吗？

YAML 配置文件是运行时加载的，修改后**不需要重新编译**。但如果之前的构建使用了 `catkin_make install`，则需要重新执行 `catkin_make install` 将新 YAML 复制到 install 目录。一般使用 `catkin_make`（不带 install）的用户无需任何额外操作。
