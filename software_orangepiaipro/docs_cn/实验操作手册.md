# Spot Micro 软件实验操作手册

> **阅读方式**：本文按"准备 → 构建 → 联调"顺序编排，适合零基础的复刻流程。依次完成每个阶段的"目标"和"操作"，遇到报错先参考对应阶段的排查提示，再进入下一阶段。
>
> **2026-03 补充说明（非常重要）**：如果你使用的是 **Orange Pi AI Pro + Ubuntu 22.04**，请不要把本手册里的旧式 `catkin_ws` 流程直接套到 ROS base 源码树上。推荐做法是：
> 1. 先在 `~/Desktop/SpotMicro/ros_noetic_ws` 里源码编译 ROS1 Noetic；
> 2. 再单独建立 `~/Desktop/SpotMicro/spotmicro_ws` 作为机器狗 overlay 工作区；
> 3. SpotMicro 包只放进 `spotmicro_ws/src`，不要混进 `ros_noetic_ws/src`。
>
> 本手册后续仍保留通用 catkin 工作区描述，便于旧环境参考；但对 Orange Pi Ubuntu 22.04，请以本补充说明和 `docs_cn/README.md` 的双工作区方案为准。

## 目录约定与终端初始化（★ 必读）

### 目录结构约定

**克隆本仓库后，你只会得到 `spotMicro-Chinese/` 目录。** 后续需要在它的**同级**创建 Catkin 工作区 `catkin_ws/`。

> **Orange Pi Ubuntu 22.04 改用下列结构：**
> ```
> ~/Desktop/SpotMicro/
> ├── ros_noetic_ws/     # ROS1 Noetic 源码基础工作区
> ├── spotmicro_ws/      # 机器狗 overlay 工作区
> └── spotMicro-Chinese/ # 本仓库，作为源码/文档仓库
> ```
> 这样可以避免把基础 ROS 与业务包混编，后续升级、清理和排错都更稳。

本手册所有命令基于一个核心约定：**`catkin_ws/` 与 `spotMicro-Chinese/` 位于同一个父目录下**。你可以把仓库克隆到任何位置，只要保持这个平级关系，文档中的相对路径命令即可直接使用。

```
<任意位置>/                       ← 父目录（你自己选择，可以是 ~、~/projects 等）
├── catkin_ws/                    ← Catkin 工作区（按阶段 B 的步骤创建）
│   └── src/
│       └── spot_micro -> ../../spotMicro-Chinese/software  （相对路径符号链接）
└── spotMicro-Chinese/            ← 本仓库（git clone 得到）
    ├── software/                 ← ROS 包源码
    ├── software_standalone/      ← 无需 ROS 的独立版
    └── hardware/                 ← 硬件资料
```

> **示例**：假设你在家目录下操作：
> ```bash
> cd ~
> git clone <仓库地址> spotMicro-Chinese
> mkdir -p catkin_ws/src      # catkin_ws 与 spotMicro-Chinese 同级
> ```
> 此时 `~/catkin_ws/` 和 `~/spotMicro-Chinese/` 就满足了平级关系。

### 每个新终端的固定操作（★ 非常重要）

**所有涉及 ROS 的操作（roscore、rosrun、rostopic 等）都要求先初始化环境。每打开一个新的 SSH 或本地终端，必须先执行以下两步：**

```bash
# 第 1 步：进入 catkin_ws 目录（请替换为你的实际路径）
cd <你的catkin_ws路径>

# 第 2 步：加载 ROS 环境 + 本项目自定义包（相对路径，无需修改）
source /opt/ros/noetic/setup.bash && source devel/setup.bash
```

也可以合并为一行：

```bash
cd <你的catkin_ws路径> && source /opt/ros/noetic/setup.bash && source devel/setup.bash
```

这条命令做了三件事：
1. 进入 Catkin 工作区目录
2. `source /opt/ros/noetic/setup.bash` — 加载 ROS 系统环境
3. `source devel/setup.bash` — 加载本项目编译出来的自定义包和消息（相对路径，已在 catkin_ws 内）

> **不执行会怎样？** 你会遇到 `rosrun: command not found`、`Package xxx not found`、`Failed to call service` 等各种报错。**每个终端窗口都是独立的环境，必须每个都执行一次。**

### 简化方案：添加 Bash 别名（推荐）

将下面命令中的 `<你的catkin_ws绝对路径>` 替换为你的真实路径后执行一次，以后每次只需输入 `spot` 即可完成初始化：

```bash
# ★ 替换 <你的catkin_ws绝对路径>，例如 ~/catkin_ws 或 /home/ubuntu/myproject/catkin_ws
echo 'alias spot="cd <你的catkin_ws绝对路径> && source /opt/ros/noetic/setup.bash && source devel/setup.bash"' >> ~/.bashrc
source ~/.bashrc
```

设置完成后，今后每个新终端只需：

```bash
spot
```

### 不涉及 ROS 的终端

以下操作**不需要** source ROS 环境，直接 cd 到对应目录即可：

- PCA9685 硬件测试脚本（如果你参照 `SERVO_PAC9685/` 目录另行搭建了测试环境）
- Standalone 独立版运行（`spotMicro-Chinese/software_standalone/`）

## 阶段 A：准备硬件与系统
**目标**：确认硬件、软件版本满足要求，并完成系统级依赖安装。

1. **硬件清单**
   - 主控：树莓派 4B(4GB) 或 Jetson Nano（本文以树莓派为例）。
   - 执行机构：PCA9685 舵机驱动板 + 12 个 PDI-HV5523MG 舵机，供电需独立 7.4V LiPo，注意与主控共地。
     > **PDI-HV5523MG 关键参数**：旋转角度 **120°**（非 180°），脉宽范围 **900–2100 µs**，中心脉宽 1500 µs，工作电压 6.0–8.4V。所有脚本和配置均按此参数设定，**更换舵机型号时必须同步修改相关参数**，否则可能导致堵转过热或烧毁。
   - 传感器（可选）：RPLidar A1、LCD 模块、IMU（仓库暂不支持，见阶段 H）。
   - 开发机：Ubuntu 20.04 x64，安装 ROS Noetic。
2. **系统依赖**
   ```bash
   sudo apt update
   sudo apt install git build-essential cmake python3-rosdep python3-vcstool libeigen3-dev \
        python3-catkin-tools ros-noetic-rosserial ros-noetic-joy ros-noetic-tf \
        ros-noetic-diagnostic-updater ros-noetic-xacro ros-noetic-rviz \
        ros-noetic-ros-control ros-noetic-ros-controllers ros-noetic-tf2-eigen \
        python3-smbus python3-matplotlib python3-empy
   sudo rosdep init   # 仅第一次
   rosdep update
   ```
   > **常见坑**：Ubuntu 20.04 默认没有 `python` 命令，若脚本使用 `/usr/bin/python`，请执行 `sudo apt install python-is-python3` 或改 shebang。
3. **账号权限**
   - 在树莓派上启用 I2C：`sudo raspi-config -> Interfacing Options -> I2C -> Enable`。
   - 将用户加入 `i2c` 与 `dialout` 组：`sudo usermod -a -G i2c,dialout $USER`，重启生效。

## 阶段 B：创建 Catkin 工作区并链接源码
**目标**：在 `spotMicro-Chinese/` 的同级目录创建 `catkin_ws/`，并用相对路径链接仓库。

> **Orange Pi Ubuntu 22.04 例外**：如果你已经在 `ros_noetic_ws` 中源码编译通过 ROS1 Noetic，请改为创建 `spotmicro_ws`，并先执行：
> ```bash
> source /home/HwHiAiUser/Desktop/SpotMicro/ros_noetic_ws/devel/setup.bash
> ```
> 然后再在 `spotmicro_ws` 中迁移和编译 SpotMicro 包。

> 以下命令假设你当前位于 `spotMicro-Chinese/` 的**父目录**（即包含仓库的那个文件夹）。

1. **初始化工作区**
   ```bash
   # 在 spotMicro-Chinese 的同级创建 catkin_ws
   mkdir -p catkin_ws/src
   cd catkin_ws
   source /opt/ros/noetic/setup.bash
   catkin init
   catkin config --extend /opt/ros/noetic --cmake-args -DCMAKE_BUILD_TYPE=Release
   ```
2. **链接仓库到 `src`**（使用相对路径，确保可移植性）
   ```bash
   cd src
   ln -s ../../spotMicro-Chinese/software spot_micro
   ```
   > **路径说明**：`../../spotMicro-Chinese/software` 是从 `catkin_ws/src/` 出发的相对路径，因此只要 `catkin_ws` 与 `spotMicro-Chinese` 保持平级关系，该链接在任何位置都能正确工作。
   > **避免递归**：切勿在 `spotMicro-Chinese/software` 内再创建指向自身的软链接，否则 `catkin build` 会报 "Multiple packages found with the same name"。
3. **忽略旧版扩展包**
   ```bash
   # 从 catkin_ws/src 出发
   cd ../../spotMicro-Chinese/software/extensions
   touch packages/CATKIN_IGNORE
   ```
   防止 `rosdep`/`catkin` 发现重复包。

## 阶段 C：首次构建与日常使用
**目标**：安装依赖、完成一次成功的 `catkin build`，并掌握日常启动命令。

1. **安装 ROS 依赖并构建**
   ```bash
   cd <你的catkin_ws路径>
   source /opt/ros/noetic/setup.bash
   rosdep install --from-paths src --ignore-src -r -y
   catkin build
   ```
   若之前迁移过工作区，遇到 "stored path ≠ commanded path" 错误，请执行 `catkin clean -y` 或 `rm -rf build devel` 后重建。

2. **★ 日常打开终端的固定动作（每个终端都要执行！）**

   ```bash
   cd <你的catkin_ws路径> && source /opt/ros/noetic/setup.bash && source devel/setup.bash
   ```

   或者使用你设置好的 `spot` 别名（见本文开头"简化方案"）。

3. **验证**
   ```bash
   # 确认构建成功（在 catkin_ws 目录下）
   catkin build    # 末尾应显示所有 9 个包 succeeded

   # 确认环境变量
   echo $ROS_PACKAGE_PATH    # 应包含 catkin_ws/src/spot_micro
   ```

## 阶段 D：按顺序启动与联调
**目标**：在硬件连接良好的前提下，使机器人能接受键盘命令并站立/行走。

> **重要**：下面每个终端都是**独立的 SSH 或终端窗口**，每个都必须先执行初始化命令。以下用 `[初始化]` 代替完整的初始化命令：
> ```bash
> [初始化] = cd <你的catkin_ws路径> && source /opt/ros/noetic/setup.bash && source devel/setup.bash
> ```
> 如果你已设置 `spot` 别名，直接输入 `spot` 即可替代 `[初始化]`。

### 终端 A：ROS Master

```bash
[初始化]
roscore
```

保持运行，不要关闭。

### 终端 B：PCA9685 驱动

```bash
[初始化]
rosrun i2cpwm_board i2cpwm_board
```

若未连接硬件会报 "Failed to open /dev/i2c-1"，属预期行为。

### 终端 C：单舵机调试/校准

```bash
[初始化]
rosrun servo_move_keyboard servoMoveKeyboard.py
```

用于逐个舵机标定，进入 `oneServo` 模式后输入通道号。

### 终端 D：键盘速度命令

```bash
[初始化]
rosrun spot_micro_keyboard_command spotMicroKeyboardMove.py
```

### 终端 E：步态控制核心

```bash
[初始化]
rosrun spot_micro_motion_cmd spot_micro_motion_cmd_node
```

### 可选终端

```bash
[初始化]

# LCD 监控
rosrun lcd_monitor sm_lcd_node.py

# 姿态可视化
rosrun spot_micro_plot spotMicroPlot.py
```

> **启动顺序**：务必按 A → B → C/D/E 的顺序，先有 `roscore`，再有驱动，最后是上层应用。若 `spot_micro_motion_cmd` 报 `Failed to call service servo_config`，说明终端 B 的 i2cpwm_board 驱动没有运行或无法访问 0x40，先排查 I2C。

## 阶段 E：舵机与 PCA9685 调试
**目标**：在接入 ROS 前确认硬件通电正常，并完成舵机映射。

1. **独立脚本快速自检**（无需 ROS，无需 source）

   本仓库 `SERVO_PAC9685/raspi_pca9685_test/` 提供了不依赖 ROS 的硬件测试脚本（详见该目录的 `README.md`）。如果你将该目录也放在与 `spotMicro-Chinese` 同级的位置，可从仓库目录出发：

   ```bash
   # 从 spotMicro-Chinese 出发
   cd ../SERVO_PAC9685/raspi_pca9685_test

   # 首次安装依赖
   sudo apt install python3-pip python3-smbus i2c-tools
   sudo pip3 install adafruit-circuitpython-pca9685 adafruit-blinka

   # 检测 PCA9685
   sudo i2cdetect -y 1        # 需看到 0x40

   # 所有舵机归中心
   sudo python3 set_all_servos.py --angle 60    # 60° = PDI-HV5523MG 中心位置

   # 单舵机测试
   sudo python3 set_servo.py 1 60    # 舵机1 (LF_1 左前肩) -> 60°
   ```

   - 若提示 `No module named 'board'` 或 `RPi`，说明 `adafruit-blinka` 未装在当前 Python；请使用系统 Python 或在 conda 里安装。
   - `PermissionError: '/dev/i2c-1'`：把用户加入 `i2c` 组或直接使用 `sudo`。
   - 测试脚本的角度范围为 `0–120`（非 0–180），已匹配 PDI-HV5523MG 舵机的 120° 行程。

2. **舵机映射（默认 12 通道）**

   接线顺序：每条腿按 **肩→大腿→小腿** 排列，腿的顺序为 **左前→右前→左后→右后**。

   | 舵机编号 | 关节 | YAML 名称 | 位置 |
   | --- | --- | --- | --- |
   | 1 | FL_HIP | LF_1 | 左前肩 |
   | 2 | FL_UPPER_LEG | LF_2 | 左前大腿 |
   | 3 | FL_LOWER_LEG | LF_3 | 左前小腿 |
   | 4 | FR_HIP | RF_1 | 右前肩 |
   | 5 | FR_UPPER_LEG | RF_2 | 右前大腿 |
   | 6 | FR_LOWER_LEG | RF_3 | 右前小腿 |
   | 7 | RL_HIP | LB_1 | 左后肩 |
   | 8 | RL_UPPER_LEG | LB_2 | 左后大腿 |
   | 9 | RL_LOWER_LEG | LB_3 | 左后小腿 |
   | 10 | RR_HIP | RB_1 | 右后肩 |
   | 11 | RR_UPPER_LEG | RB_2 | 右后大腿 |
   | 12 | RR_LOWER_LEG | RB_3 | 右后小腿 |

   此映射已与 `spot_micro_motion_cmd.yaml` 和 `default.yaml` 的 `num` 字段保持一致。若重新布线，请同步修改上述 YAML 文件中对应的 `num` 值。

   > **⚠ PDI-HV5523MG 角度范围注意**：本项目舵机总行程为 **120°**（0°–120°），物理中心为 **60°**。在 `servo_move_keyboard` 标定或手动测试时，PCA9685 有效 PWM 范围约为 **184–430**（对应 900–2100 µs），中心值约 **307**（对应 1500 µs）。**切勿将 PWM 设到此范围之外**，否则超出舵机物理极限会造成堵转损坏。

3. **使用 `servo_move_keyboard` 记录参数**

   需要先在**两个终端**中分别启动 ROS 环境：

   ```bash
   # ── 终端 1 ──
   [初始化]
   rosrun i2cpwm_board i2cpwm_board

   # ── 终端 2 ──
   [初始化]
   rosrun servo_move_keyboard servoMoveKeyboard.py
   ```

   - 进入 `oneServo` 模式输入通道号，使用 `f/g/j/k` 微调并记录中心/极值。
   - 将数据填入 `docs_cn/舵机校准参考表格.ods`，再写入 YAML。

## 阶段 F：RPLidar（无图形环境）
**目标**：在纯终端环境下验证 Lidar 数据是否发布。

1. 安装驱动：`sudo apt install ros-noetic-rplidar-ros`
2. 确认串口：插上 USB，运行 `ls /dev/ttyUSB*`（例如 `/dev/ttyUSB0`）。如权限不足，`sudo usermod -a -G dialout $USER` 并重启。
3. 启动（新终端）：
   ```bash
   [初始化]
   roslaunch rplidar_ros rplidar_a1.launch serial_port:=/dev/ttyUSB0 frame_id:=laser
   ```
4. 新终端验证：
   ```bash
   [初始化]
   rostopic hz /scan
   rostopic echo /scan
   ```
   频率约 8~10 Hz 即为正常。如需可视化，可 `rosbag record /scan` 并在有图形的电脑用 RViz 打开。

## 阶段 G：常见问题速查

| 现象 | 可能原因 | 对策 |
| --- | --- | --- |
| `rosrun: command not found` | 当前终端未执行初始化 | 执行 `[初始化]` 或 `spot` 别名 |
| `Package xxx not found` | 当前终端未 source `devel/setup.bash` | 同上 |
| `catkin build` 报 "stored path ≠ commanded path" | 搬动了工作区但未清理 build/devel | 在 catkin_ws 下执行 `rm -rf build devel` 或 `catkin clean -y` 后重建 |
| `Multiple packages found with the same name` | 存在自指软链接或 extensions 未忽略 | 删除 `src/spot_micro` 重新 `ln -s ../../spotMicro-Chinese/software spot_micro` 并在 `extensions/packages` 放 `CATKIN_IGNORE` |
| `rosrun ... Found but not executable` | 脚本缺执行权限 | 在 catkin_ws 下执行 `chmod +x src/spot_micro/<package>/scripts/*.py && catkin build <package>` |
| `Failed to open /dev/i2c-1` | 未启用 I2C、无权限、硬件未接 | 启用 I2C、加入 `i2c` 组、检查供电与 `i2cdetect -y 1` |
| `spot_micro_motion_cmd` 提示缺少 `tf2_eigen` | 未安装依赖 | `sudo apt install ros-noetic-tf2-eigen` 后重新编译 |
| `rplidar_ros` 找不到 launch | 未安装 `ros-noetic-rplidar-ros` 或包路径未 source | 安装驱动并重新执行初始化 |
| `Failed to call service servo_config` | PCA9685 驱动未运行或启动顺序错误 | 按阶段 D 顺序重新启动，确保 i2cpwm_board 正常发布话题 |

更多构建细节可参考 `software/docs/build_notes.md`。

## 阶段 H：功能边界与扩展
- **MPU-6050**：目前仓库仅在文档中提及该 IMU，未提供 ROS 驱动或控制器接口。如需 IMU，请自行集成 `ros-noetic-imu-filter-madgwick` 等包，并修改控制器逻辑以消费 `sensor_msgs/Imu`。
- **RPLidar ≠ 自动导航**：当前仅提供 `/scan` 数据，未集成 SLAM/定位/避障。要实现自主行走，需要额外搭建 gmapping/hector/cartographer 等导航栈，并将其输出的 `/cmd_vel` 接入 `spot_micro_motion_cmd`。
- **ROS 驱动 vs. 独立脚本**：`ros-i2cpwmboard` + `servo_move_keyboard` 负责 ROS 话题、12 关节映射；`SERVO_PAC9685/raspi_pca9685_test` 提供裸脚本便于批量位姿测试。建议先用脚本验证硬件，再进入 ROS 流程进行标定与步态调试。
