# 恢复流程

## 1. 恢复目标

目标是在另一台香橙派，或者当前香橙派系统损坏重装后，恢复出以下两层结构：

- `~/Desktop/SpotMicro/ros_noetic_ws`
- `~/Desktop/SpotMicro/spotmicro_ws`

## 2. 恢复前提

先准备：

- Ubuntu 22.04 基础系统
- Git
- 已启用网络
- 已准备好备份仓库或导出的压缩包

## 3. 拉取备份仓库

```bash
cd ~/Desktop/SpotMicro
git clone <你的远程仓库地址> orangepi_ros_backup
cd orangepi_ros_backup
chmod +x scripts/*.sh
```

## 4. 安装依赖

先参考：

- `manifests/apt-manual.txt`
- `manifests/notes.txt`
- `docs/香橙派环境注意事项.md`

至少优先确认：

```bash
sudo apt update
sudo apt install -y python3-defusedxml libi2c-dev
```

## 5. 恢复 ROS base workspace

```bash
mkdir -p ~/Desktop/SpotMicro/ros_noetic_ws/src
cp -a workspaces/ros_noetic_ws/src/. ~/Desktop/SpotMicro/ros_noetic_ws/src/
cd ~/Desktop/SpotMicro/ros_noetic_ws
catkin_make --force-cmake -DCATKIN_ENABLE_TESTING=OFF -DPYTHON_EXECUTABLE=/usr/bin/python3
```

如果你备份的是压缩包，也可以：

```bash
mkdir -p ~/Desktop/SpotMicro/ros_noetic_ws
tar -xzf archives/ros_noetic_ws_src_*.tar.gz -C ~/Desktop/SpotMicro/ros_noetic_ws
cd ~/Desktop/SpotMicro/ros_noetic_ws
catkin_make --force-cmake -DCATKIN_ENABLE_TESTING=OFF -DPYTHON_EXECUTABLE=/usr/bin/python3
```

## 6. 恢复 SpotMicro overlay workspace

```bash
mkdir -p ~/Desktop/SpotMicro/spotmicro_ws/src
cp -a workspaces/spotmicro_ws/src/. ~/Desktop/SpotMicro/spotmicro_ws/src/
source ~/Desktop/SpotMicro/ros_noetic_ws/devel/setup.bash
cd ~/Desktop/SpotMicro/spotmicro_ws
catkin_make -DCATKIN_ENABLE_TESTING=OFF -DPYTHON_EXECUTABLE=/usr/bin/python3
```

## 7. 配置 shell 环境

建议按顺序写入：

```bash
echo 'source ~/Desktop/SpotMicro/ros_noetic_ws/devel/setup.bash' >> ~/.bashrc
echo 'source ~/Desktop/SpotMicro/spotmicro_ws/devel/setup.bash' >> ~/.bashrc
source ~/.bashrc
```

## 8. 恢复完成后校验

```bash
roscore
```

新开一个终端：

```bash
source ~/Desktop/SpotMicro/ros_noetic_ws/devel/setup.bash
source ~/Desktop/SpotMicro/spotmicro_ws/devel/setup.bash
rospack find roscpp
rospack find i2cpwm_board
rospack find spot_micro_motion_cmd
```

## 9. 常见问题快速处理

### 9.1 roscore 报 defusedxml 缺失

```bash
sudo apt install -y python3-defusedxml
```

### 9.2 catkin_make 误用了 Conda 的 python3

```bash
conda deactivate
unset PYTHONPATH
unset PYTHONHOME
which python3
```

确认输出是 `/usr/bin/python3` 后再编译。

### 9.3 gmock / gtest 相关错误

重新构建时明确关闭测试：

```bash
catkin_make -DCATKIN_ENABLE_TESTING=OFF -DPYTHON_EXECUTABLE=/usr/bin/python3
```

### 9.4 I2C 库找不到

先确认：

```bash
ls /usr/lib/aarch64-linux-gnu/libi2c.so*
dpkg -L libi2c-dev | grep libi2c
```

如果缺失则安装：

```bash
sudo apt install -y libi2c-dev
```
