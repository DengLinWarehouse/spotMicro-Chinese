# 依赖清单维护

## 1. 目标

这个目录用于保存香橙派当前环境的依赖快照，便于以后重装系统或迁移到新机器时快速复现。

## 2. 推荐维护的清单

建议至少保留以下文件：

- `manifests/system-info.txt`
- `manifests/apt-manual.txt`
- `manifests/python3-freeze.txt`
- `manifests/ros-environment.txt`
- `manifests/notes.txt`

## 3. 建议记录的系统信息

- Ubuntu 版本
- 内核版本
- Python 解释器路径
- `python3 --version`
- `gcc --version`
- `cmake --version`
- `git --version`

## 4. 特别要记录的依赖

结合本次实际部署，建议重点关注：

- `python3-defusedxml`
- `libi2c-dev`
- `python3-catkin-pkg`
- `python3-empy`
- `python3-nose`
- `python3-setuptools`
- `python3-yaml`
- `python3-rospkg`
- `python3-rosdep`
- `python3-vcstool`
- `libeigen3-dev`
- `libtinyxml-dev`
- `libtinyxml2-dev`
- `liblog4cxx-dev`

## 5. 推荐做法

每次环境稳定后执行一次：

```bash
bash scripts/capture_manifests.sh
```

如果你安装了新的系统依赖，最好马上重新执行一次，并提交到 Git。

## 6. notes.txt 建议手工补充的内容

建议把下面这些非自动化信息写入 `manifests/notes.txt`：

- 当前设备型号
- 当前系统镜像来源
- 是否启用了 I2C
- PCA9685 使用的 I2C 总线号
- `~/.bashrc` 里 ROS 的 source 顺序
- 本地做过哪些补丁
- 哪些包来自 `extensions/packages`
