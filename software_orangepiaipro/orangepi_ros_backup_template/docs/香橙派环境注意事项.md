# 香橙派环境注意事项

## 1. 双工作区必须保持分离

实际验证后，强烈建议始终使用：

- `ros_noetic_ws`：只放 ROS1 Noetic 基础源码
- `spotmicro_ws`：只放 SpotMicro 业务包

不要把业务包直接混入 `ros_noetic_ws/src`。

## 2. Conda 不要参与 ROS 构建

如果终端里出现 `(base)`，要先警惕当前 Python 解释器是否来自 Conda。

构建前至少确认：

```bash
which python3
/usr/bin/python3 -c "import sys; print(sys.executable)"
```

## 3. 编译优先使用系统 Python

统一推荐：

```bash
catkin_make -DCATKIN_ENABLE_TESTING=OFF -DPYTHON_EXECUTABLE=/usr/bin/python3
```

## 4. roscore 能启动才算 base 环境真正可用

仅仅 `catkin_make` 成功还不够，必须继续验证：

- `roscore`
- `rospack find roscpp`
- `echo $ROS_PACKAGE_PATH`

## 5. 本次实际遇到的重要问题

### 5.1 `python3-defusedxml` 缺失

表现：`roscore` 启动时报 `ModuleNotFoundError: No module named 'defusedxml'`

处理：

```bash
sudo apt install -y python3-defusedxml
```

### 5.2 gmock/gtest 触发配置失败

表现：`/usr/src/gmock` 不存在或测试配置失败。

处理：关闭 catkin testing。

### 5.3 SpotMicro 某些包需要 Python 3 修正版脚本

例如：

- `spot_micro_keyboard_command`
- `servo_move_keyboard`
- `spot_micro_plot`
- `lcd_monitor`

优先参考 `extensions/packages` 中的修正版内容。

### 5.4 motion_cmd 需要兼容性补丁

本次实际处理过：

- C++17 相关问题
- `tf2_eigen` 相关 include/转换问题

所以恢复后如果源码版本不同，不要默认认为一定能一次编译通过。

## 6. `.bashrc` source 顺序

建议固定为：

```bash
source ~/Desktop/SpotMicro/ros_noetic_ws/devel/setup.bash
source ~/Desktop/SpotMicro/spotmicro_ws/devel/setup.bash
```

先 base，后 overlay。

## 7. 建议保留的人工记录

除了自动脚本输出，还应长期维护：

- 当前稳定版本对应的 commit
- 实际可运行的 launch 命令
- PCA9685 所接 I2C 总线号
- 舵机编号和标定数据位置
- 哪些文件做过手工修改
