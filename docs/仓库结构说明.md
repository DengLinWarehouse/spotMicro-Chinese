# 仓库结构说明（2026-03-12）

## 顶层目录速览
- `软件部分/`：镜像自官方 `spotMicro` 仓库的 ROS 工作区，详细说明见 `软件部分/LOCALIZATION_LAYOUT.md`。
- `软件部分/docs_cn/`：中文教材、学习指引与校准表格，保持与镜像主体解耦。
- `软件部分/extensions/`：原本的本地化 ROS 包与顶层 CMake 覆盖文件全部备份于此，按需在 overlay 中启用。
- `software_standalone/`：纯 Python 版本的独立控制器，结构保持原样，建议在虚拟环境中运行；其测试缓存仍位于 `software_standalone/.pytest_cache`，如需清理可直接删除。
- `docs/assets/`：仓库公共文档使用的图像资源（已从旧的 `assets/` 目录迁移至此）。
- `docs/`：本文件及后续中文文档的统一入口。
- `local_cache/pip_cache/`：原 `.tmp` 目录中的 pip wheel 与 build 缓存已移动在此，防止污染主目录。
- `硬件部分/`：硬件物料清单与装配指南，当前按用户要求未做改动。

## 使用建议
1. **同步流程**：在 `软件部分/` 中开发时保持镜像完整，如需自定义修改请复制到 `软件部分/extensions/` 或外部 overlay。
2. **文档与素材**：仓库层面的 README 默认引用 `docs/assets/`，撰写新文档时也请将图片放入该目录以避免重复。
3. **临时文件**：新的缓存路径集中在 `local_cache/`，后续临时数据请统一放入该目录，便于批量清理。
4. **独立控制器**：`software_standalone/` 建议使用 `python -m venv` 创建虚拟环境后再执行 `pip install -r requirements.txt`，以免污染全局 Python。

如需新增模块或文档，请在对应子目录添加并记录在此文件中，保持结构清晰，方便团队成员快速定位资源。
